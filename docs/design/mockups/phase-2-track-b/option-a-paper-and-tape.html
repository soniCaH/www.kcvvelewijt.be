Paper & tape.

Tests the hypothesis that even functional UI chrome adopts the editorial paper feel — cream surfaces, hard offset shadows, sub-degree rotation, tape accents, mono caps. Atoms read as physical paper artefacts even when they are functional controls. Argument FOR: visual consistency with TapedCard, EditorialHeading, and PullQuote. Argument AGAINST: chrome wants to disappear, not draw editorial weight; risk of competing with editorial primitives for attention.

★ 01 · SPINNER — halftone-dot cycle + torn-ticket ephemera

Halftone kringloop.

Eight ink-stamp dots arranged around a paper circle, each fading sequentially at 0.15s offsets. Reads as silkscreen / risograph print texture. Composes the same dot vocabulary as the perforation column on <TicketStub>.

★ 02 · BRANDED TABS — paper-card strip, jersey tape on active

Geplakte tabbladen.

Each tab is a flat paper card (<TapedCard> vocabulary — border-2 ink + shadow-paper-sm, no rotation on inactive — chrome stays straight to read as a row). Active tab inverts to ink + cream and gets a jersey washi tape strip pinned to the top-left corner at -8°.

↑ FOR

Tabs read as a printed programme's section dividers — physically recognisable. The jersey tape strip on active is unmistakable ("torn from the supporters' programme"). Reuses <TapeStrip> + <TapedCard> vocabulary.

↓ AGAINST

Five card-shadows + a tape strip = visual weight that may compete with adjacent editorial content. Inactive tabs still carry full border + shadow; reading them as "navigation chrome" rather than "another row of cards" depends on context.

★ NOTES — token map, primitive composition, prop preservation

Token map

SURFACE / TEXT

  • Tab + chip body: --color-cream default, --color-cream-soft for variation
  • Active tab: --color-ink bg + --color-cream text
  • Active chip: --color-jersey bg + --color-ink text
  • Body text + count badges: --color-ink on cream
  • Spinner dots: --color-jersey-deep

STRUCTURE / DEPTH

  • Every atom: border: 2px solid var(--color-ink)
  • Every atom: --shadow-paper-sm (4×4 ink offset, no blur)
  • Press state: shadow collapses to 3×3 + translate +1/+1
  • Tape strips: --color-jersey/jersey-deep stripe pattern, -8° to -10°
  • Match card rotation: --rotate-tape-a..d (sub-degree pool)

Primitive composition map

  • Spinner cells compose <TapedCard> (border-2 ink + shadow-paper-sm + bg cream); halftone dots are the same dot vocabulary as <TicketStub> perforations.
  • Spinner sub-variant A.2 (torn ticket) directly composes <TicketStub> — perforated edge + ink-stamp label.
  • BrandedTabs active state composes <TapedCard bg="ink"> (inverted) + <TapeStrip color="jersey" position="tl" rotation="-8">.
  • FilterTabs chips are <TapedCard> at chip scale (border + shadow); count badges use <MonoLabel variant="pill-ink"> sizing (11px / 0.08em / pill padding).
  • HorizontalSlider arrows compose <TapedCard> (square 48px) + <TapeStrip color="jersey"> at -10°. Glyph is typographic Freight Display italic per the "no Lucide where the glyph reads" preference.
  • Cohesion mini-mockup uses <EditorialHeading> period-terminated heading (Wedstrijden.) + <MonoLabelRow> kicker.

Prop-surface preservation

Every atom's TypeScript surface is preserved. Visual changes are token + layout only. Specifically:

  • Spinner: size: "sm" | "md" | "lg" | "xl", variant: "primary" | "secondary" | "white" | "logo", label. Halftone implementation replaces the SVG circle for non-logo variants. Sizes 16/32/48/64 px unchanged.
  • BrandedTabs: tabs, activeTabId, onTabChange, ariaLabel?. No new props.
  • FilterTabs: tabs, activeTab, onChange?, size, showCounts?, icon?, renderAsLinks?. Icon prop type swap to Phosphor Icon per Phase 2.A.6 — Direction A is compatible with that swap.
  • HorizontalSlider: children, title?, theme: "light" | "dark". No new props.
  • ScrollArrowButton: direction, onClick, variant: "light" | "dark". May need size?: "sm" | "md" if compact slider contexts surface (see Slider trade-offs).

Direction-level trade-offs

↑ DIRECTION FOR

Maximum coherence with editorial primitives. The site reads as one physical artefact — programme, scarf, ticket — without exception. Every chrome decision reinforces the core thesis.

↓ DIRECTION AGAINST

Cumulative paper-shadow density on chrome-heavy surfaces (filter + tab + slider arrow + spinner all in one viewport) may compete with editorial cards for visual weight. Editorial primitives already earn paper depth; chrome carrying it too can flatten the hierarchy.
