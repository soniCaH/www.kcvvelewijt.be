QuotesBlock — sourcing & pairing

Last per-element drill. §5.3 specs a 2-card pair (ink + cream PullQuote). With RecentMatchesGrid dropped at 6.d7, QuotesBlock is the only remaining home for the parked 6.d4 dark-band aesthetic — its tone="ink" card.

Sourcing options A/B extend the 6.d5 bio PT decorator pattern OR introduce a new schema field. Variant A is "intuitive + least invasive" (same rationale that won 6.d5). Variant C halves the pair to a single quote. Variant D drops entirely and retires the dark band along with it.

Cross-age · privacy: no quotes from minors
Variant A

Extend bio decorators.

Same pullquote PT decorator from 6.d5. Bio paragraph can carry up to 3 marked spans: span #1 → BioBlock inline; span #2 → QuotesBlock left (ink); span #3 → QuotesBlock right (cream). Single source of truth. Zero new schema fields.

Data sourcing

  • Reuses 6.d5 lock: pullquote PT decorator on player.bio
  • Renderer: scan bio for decorator spans, index by position
  • BioBlock: first span (already locked)
  • QuotesBlock: next 2 spans (this lock)

Tradeoff

  • Cleanest reuse — extends an existing decision rather than introducing new vocabulary
  • Δ Requires the bio to be long enough + the editor to mark 3 spans. Short bios collapse the section gracefully.
  • Privacy: minors don't get bios → no leakage
  • All quotes are "from the bio" — quotes feel narrational rather than spoken-in-interview
Variant B

New featuredQuotes[] field.

New schema: player.featuredQuotes: array<{text, source}> (length 0-2). Quotes independent of bio — can quote interview, training-day chat, social post. Each entry has its own attribution. Maximum editorial flexibility.
