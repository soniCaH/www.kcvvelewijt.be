RecentMatchesGrid — per-player attribution vs team-result vs drop

Canonical mockup shows four ink cards: date · competition · scoreline · per-player goal / assist tag · venue. The per-player attribution is what makes this the most-expensive section in Phase 6.A — PSD's goalsScored arrays are team-global event lists, not keyed to a single player, so attribution requires either a new BFF endpoint or a join across goal events + appearances.

Each column shows the A-team case and the U13 case (where the section auto-hides because PSD likely doesn't publish per-player youth match data). Variant D matches the cumulative trend across 6.d3 / 6.d4 / 6.d6 — scope discipline over mockup fidelity.

Cost class: NEW BFF work, not editorial

Variant D

Drop RecentMatches.

No recent-matches surface on the player profile. The player's team-detail page already has match schedule + standings. Linking to that page from the profile is enough. Honors 6.d0 drop-section escape. Cumulative drop-section count: 4.
