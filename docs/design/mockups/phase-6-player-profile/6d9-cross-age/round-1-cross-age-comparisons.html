Cross-age section availability + privacy for minors

Last design-domain drill. Per [[project_player_profile_all_ages]], the player profile renders ALL players: A-team adults through U6 toddlers. Sections auto-hide when data is absent (BioBlock, QuotesBlock) but the hero ALWAYS renders something — what should it show, and what's the privacy treatment for minors?

Three representative players rendered per variant — Maxim (A-team, 26), Senne (U17, 16), Lars (U8, 7) — to see how the variants treat the full age range, not just the extremes.

A-team Maxim, 26 — Aanvaller — bio + 2 marked spans U17 Senne, 16 — Middenvelder — no bio U8 Lars, 7 — minor — no bio
Variant A

Uniform — treat everyone the same.

Same shape, same fields, same photo treatment for every age. Data-driven hide handles BioBlock + QuotesBlock. Hero always shows full meta (DOB + height + weight + nationality) and psdImage when present. No special minor handling.

Simplest implementation ⚠ Minor-data exposure

Treatment rules

  • Full DOB shown for everyone
  • Height + weight shown for everyone (where present)
  • psdImage rendered for everyone (where present)
  • Sections hide only on absent data

Tradeoff

  • Simplest possible implementation (one rendering path)
  • ⚠ Renders a 7-year-old's full DOB + body metrics + face publicly. Federation-level consent is not website-level consent.
  • No editorial workflow for opting out an individual minor
Variant B

Privacy-graded for minors.

Minors (under 18) get reduced metadata: age + year instead of full DOB; no height / weight for U-divisions under 13. Photo still renders when present. BioBlock + QuotesBlock handled by editorial policy (don't write bios for minors → auto-hide).

Age-gated render rules Photos still render

Treatment rules

  • Adults (≥18): full DOB + height + weight
  • U13-U18 minors: age + year only ("16 jaar · '09"); height shown; weight hidden
  • U6-U12 minors: age + year only; no height / weight
  • Photo rendered when present (no opt-out gate)
  • BioBlock + QuotesBlock auto-hide for minors via editorial policy (don't author bios for minors)

Tradeoff

  • Sane default privacy treatment without an explicit opt-out workflow
  • Δ Requires age-derivation at render time + conditional meta-row rendering
  • Photo still public — relies on PSD's federation consent
Variant C

Strict — illustration for all minors.

Above B, plus: minors never see psdImage on the hero. Default to <PlayerFigure> illustration fallback for anyone under 18. Adults render photo as before. Federation-consent doesn't carry to public website context for minors.

Photo gate Privacy-first

Treatment rules

  • Same age-gated meta as B
  • Minors: always render PlayerFigure illustration (the 6.d2 lock fallback) regardless of psdImage presence
  • Adults: psdImage as canonical, illustration only when missing
  • Photo gate driven by age < 18 at render time

Tradeoff

  • Strongest privacy stance — no public-facing minor photos
  • All minor profiles render identical illustration → low individual identity
  • Closes a real risk (PSD photo + DOB on a public URL constitutes identifiable PII)
  • An opt-IN workflow (parent / coach explicitly approves photo) can be added later
Variant D

Minimal — name + team for minors.

Strictest interpretation. Minors render hero with name + team label + jersey number only. No DOB, no height, no weight, no photo, no figure column at all. Single-column hero. Adults unchanged. The page is essentially a directory entry for U-divisions.

Minimal-for-minors Lowest fidelity for youth

Treatment rules

  • Adults (≥18): full hero unchanged
  • Minors (<18): single-column hero with name + jersey number + team only. No DOB, no height/weight, no figure column
  • Equivalent to a public directory entry — exists to confirm membership, not to feature the player

Tradeoff

  • Smallest PII surface area possible while still having a page
  • Youth profiles read as placeholders — could feel disrespectful ("their page is empty")
  • Easiest to defend to a privacy regulator
  • Photo + DOB never exposed for any minor regardless of psdImage availability
