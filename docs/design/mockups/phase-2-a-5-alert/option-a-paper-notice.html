PHASE 2 · TRACK A · ALERT · DIRECTION A ★ PAPER NOTICE 2026-04-30

Een officieel bericht.

Alerts as official printed notices. Same paper-card chrome the rest of the redesign already speaks — border-2 ink + shadow-paper-sm + tinted soft bg per semantic + sharp corners. The Phosphor Fill icon sits upper-left in an ink-bordered "stamp" box like a postmark, so the variant reads at a glance without polluting the title row. Mono caps title in pure ink, body in Inter. No tape, no rotation — alerts read as a flat sibling of BrandedTabs / FilterTabs / arrows, not as a tilted scrap on top of them.

↑ FOR

Same chrome scales cleanly to form-row width — only padding, stamp, and body size change. The notice never overpowers a 480px-wide form. Below-input placement is the canonical accessibility pattern (aria-describedby) and matches the visual weight expected in a form.

↓ AGAINST

Inline scale still carries a 36×36 stamp + 2px ink border + paper shadow — heavier than a stripped one-line helper text. If a project uses many alerts per form, density adds up. Counter-pattern: keep field-level guidance as plain helper text; reserve <Alert> for actionable state changes (validation failed, submit succeeded).

★ 04 · TRADEOFFS — direction-level summary across all variants

Waarom deze richting.

Direction A maximises reuse of the locked chrome vocabulary. The alert is structurally indistinguishable from a tab, an arrow button, or a match card — only the tinted bg + stamp glyph + mono title carry the "this is a notice" reading. Lowest-risk direction; closest neighbour of the existing system.

↑ FOR

Maximum chrome reuse — every primitive is already on main: border-2 ink, --shadow-paper-sm, cream + cream-soft bgs, mono caps title, sharp corners, the press idiom. Adds three new tokens (--color-alert-soft, --color-warning-soft, and a jersey-deep success tint) and one new structural element (the stamp box). Reads as a flat sibling of BrandedTabs and FilterTabs — exactly what the rejection brief asked for. Scales cleanly from page-level banner down to inline form error. Phosphor Fill icon stays in production code (mockup uses Unicode glyphs as stand-ins). No tape, no rotation, no editorial drama — alerts behave like reliable, official notices.

↓ AGAINST

Visually quiet by design — does not stand out the way a torn-tape or rotated alert would. The stamp + dismiss combination shows two ink-bordered boxes in one notice; on very narrow widths this can read busy. Three new soft-bg tints expand the colour token map (mitigation: they are semantic-only, never used as chrome fill). For pages with heavy chrome already on screen — many tabs + filter chips — adding multiple alerts compounds the "field of paper rectangles" effect; advisory placement should prefer one alert per view. Trades editorial novelty for system cohesion.

★ NOTES — token map, structure decisions, production handoff

Resolved

  • Alert chrome inherits paper-card vocabulary verbatim — border-2 ink + --shadow-paper-sm + sharp corners + cream paper foundation
  • Variant differentiation is bg tint + stamp glyph colour only — body and title typography are invariant
  • Stamp lives upper-left in a 44×44 ink-bordered cream box (inline scale: 36×36) — the icon never inlines with the title baseline
  • Title uses var(--font-mono) uppercase, 12px, 700 weight, in pure --color-ink — never tinted to match the variant
  • Body uses var(--font-body) in --color-ink-soft, 15px (12px inline)
  • var(--font-display) is reserved for editorial moments — the Alert never reaches for Playfair
  • Dismiss button is a paper-card mini (28×28, ink-bordered) at top-right — same press idiom as BrandedTabs hover
  • Inline scale: padding 12px 16px 12px 60px, stamp 36×36, body 14px — same component, scaled prop
  • No tape, no rotation, no border-radius — alerts are flat siblings of the chrome atoms, never editorial scraps
  • Phosphor Fill icons in production: CheckCircleFill (success), WarningFill (warning), XCircleFill (error). Mockup uses Unicode glyphs (✓ ▲ ⊗) as stand-ins

Open

STRUCTURE / DEPTH

  • Notice shadow: --shadow-paper-sm (4×4 ink)

  • Dismiss press: shadow → 2×2 + translate(1px, 1px)
  • Stamp glyph colour: matches semantic — jersey-deep / warning / alert
  • Sharp corners: no border-radius anywhere
  • No rotation: transform: none — explicit deviation from TapedCardGrid
