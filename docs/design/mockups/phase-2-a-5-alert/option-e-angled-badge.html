PHASE 2 · TRACK A · DIRECTION E ★ ANGLED BADGE 2026-04-30

Een schuin stempeltje.

A fifth direction sketched by the owner that breaks the A–D paper-card lineage. The alert is no longer a card — it's a kicker tag next to an editorial italic line. A slight-angle ink-bordered badge on the left holds the icon + mono caps label; the message lives to the right in Freight Display italic, tinted to the variant accent. No surrounding card body. No soft tinted bg. The page colour shows through. Compact, personality-driven, fanzine-coded.

★ 01 · STANDALONE VARIANTS — success · warning · error, each at natural width on the page bg

Drie varianten, één gebaar.

All three variants stacked on the page bg. Each alert occupies only the width of its content — never full-width. The badge rotation is fixed at -2deg; the italic message is upright (italic is the slant, rotation would be too much).

SUCCESS — jersey-deep badge · jersey-deep italic
✓ MELDING Bericht verzonden.
WARNING — warning-amber badge · warning-amber italic
▲ WAARSCHUWING Wedstrijd kan uitgesteld worden.
ERROR — alert-red badge · alert-red italic (canonical owner sketch)
! FOUT Geen geldig telefoonnummer.

This is the rendering the owner sketched: small dark-red badge, cream interior glyph + mono caps, slightly angled, with the italic Freight Display message in matching dark-red beside it. Treat as the canonical for this direction.

★ 02 · LONGER MESSAGE — what happens when the italic line wraps

Wanneer het uitloopt.

The design BIAS is a short headline-style message. When that guarantee can't be honoured, the italic wraps to a second line beside the badge. The badge stays vertically centred on the full message block. It works — but worth surfacing because this is where the personality starts to fight the readability, and is one of the honest tradeoffs of the direction.

2-LINE — success variant, deliberately uncomfortable case
✓ MELDING Je bericht is verzonden — we nemen binnen 2 werkdagen contact met je op.

Two lines is the practical ceiling. Beyond that the badge metaphor breaks down — the kicker tag stops feeling like a stamp and starts feeling like a column header for a paragraph it can't structure. Copy guidance for this direction must enforce headline-length messages.

★ 03 · IN-CONTEXT (FORM) — error variant under a form input, the natural fit

Waar deze schittert.

The compact form factor and headline-italic message fit inline-validation perfectly. Under a single field, beside a short error reason, the alert reads like a magazine pull-quote rather than a chrome notification. This is the home turf of Direction E.

CONTACT — phone field with invalid input
! FOUT Geen geldig telefoonnummer.

The badge anchors the eye exactly where validation matters (left edge, just under the field), while the italic carries the human-readable reason. No card body; the field's own border + the alert's red ink badge are the visual frame.

★ 04 · TRADEOFFS — honest accounting of what this direction wins and sacrifices

Wat het kost.

Direction E is the most personality-driven option of the five but pays for it with reduced information capacity, an awkward dismiss slot, and a different shape than typical alerts — which means anywhere the alert grid currently assumes full-width chrome will need rethinking.

↑ FOR

Strongest editorial voice of any of the five directions — reads as a fanzine kicker + pull-quote rather than chrome. Compact: occupies only the width it needs, never demands a full row. Pairs naturally with form-field validation where short, specific messages are the norm. Badge metaphor makes the variant unmistakable at a glance — accent colour flips badge bg AND italic tint, so the alert reads from across the page without needing to parse text. Reuses existing tokens (alert / warning / jersey-deep) and vocabulary (border-2 ink, mono caps, italic display, hard offset shadow) — nothing new in the foundation, only a new composition of approved primitives.

↓ AGAINST

Limited information capacity — can't hold multi-paragraph error explanations. Copy guidance must enforce headline-length messages, which existing alert content may violate. No obvious dismiss slot — adding a small × glyph after the message reads as an afterthought; the badge metaphor doesn't naturally hold a close affordance. Different shape from typical full-width alert chrome, so any layout that currently reserves a full row for an alert needs to be re-evaluated (the alert is now intrinsically content-width, not layout-width). Two-line wrap is the practical ceiling — beyond that the badge stops feeling like a stamp and starts feeling like a misaligned column header. Variant tint (alert-red on cream, warning-amber on cream) reduces non-text contrast vs a paper-card body; the message colour is the first thing to scrutinise on accessibility audits.

★ NOTES — token map, design constraints, decisions remaining

Decisions captured

  • Badge rotation locked at -2deg. Italic message stays upright (italic is the slant)
  • Badge corners are border-radius: 0 — sharp, hand-stamped feel
  • Badge shadow is 3×3 ink (slightly softer than the standard 4×4 paper-shadow because the badge is smaller)
  • Alert is layout-only — flex row with gap, no surrounding paper-card body, no soft tinted bg
  • Variant accent flips BOTH the badge bg AND the italic message colour — single source of variant truth
  • Mono caps badge label uses var(--font-mono); italic message uses var(--font-display) italic (Playfair stand-in for Freight Display)
  • Phosphor Fill icon mapping in production: CheckCircle (success), Warning (warning), WarningCircle (error). This mockup uses Unicode stand-ins: ✓, ▲, ! — distinct per variant per owner feedback (the earlier draft mistakenly used ▲ for both warning and error)

Open

  • Warning variant: keep bg-warning (amber) or flip to bg-ink if amber-on-cream feels too soft? Owner to confirm in cohesion check
  • Dismiss affordance: leave out entirely (current default), or add a small × in --color-ink-muted after the message? Acknowledged as an awkward fit for the badge metaphor either way
  • Maximum message length: enforce in copy guidance, or allow 2-line wrap as the ceiling? 3+ lines breaks the metaphor

Token map

BADGE / TYPOGRAPHY

  • Badge bg (success): --color-jersey-deep
  • Badge bg (warning): --color-warning
  • Badge bg (error): --color-alert
  • Badge text + glyph: --color-cream
  • Badge border: 2px solid var(--color-ink)
  • Badge label: var(--font-mono) · 11px · 700 · 0.12em tracking · uppercase
  • Message: var(--font-display) italic · 22px · 400
  • Message colour echoes badge accent — jersey-deep / warning / alert

STRUCTURE / GEOMETRY

  • Badge rotation: -2deg via --rotate-badge
  • Badge shadow: 3px 3px 0 0 var(--color-ink) (softer than the 4×4 paper-shadow because the badge is smaller)
  • Badge corners: border-radius: 0
  • Layout: display: inline-flex · gap: 18px · align-items: center
  • No card body, no tinted bg — page bg shows through
  • Max-width: 640px standalone; wraps gracefully to 2 lines under tighter constraints
