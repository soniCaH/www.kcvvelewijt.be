★ Q3 · REMAINING VERIFICATIONS — pre-lock — sponsor strip · PosterWordmark band · render scope · mobile collapse

Vier verificaties voor de lock.

Each of the four below has an obvious answer given the locks already in place. Confirm them as a batch (or push back on any single one to drill it visually) and we move to writing option-x-footer-detail.html + footer-locked.md.

1 · Sponsor strip stays separate above the footer (homepage only)

Existing SponsorsSection renders on the homepage in document flow above the footer (per Q0 audit). The locked footer (H3 + I2) doesn't host sponsor logos in its directory — instead the Doe mee column includes a Word sponsor link to /sponsors. Recommendation: keep separate. No structural change to SponsorsSection; no integration; no overlap. Section indexes (/nieuws, /evenementen, etc.) don't render the sponsor strip — sponsors stay homepage-exclusive.

2 · <PosterWordmark> band — absorbed by H3's modest wordmark

Master design §5.1 step 11 referenced a separate <PosterWordmark> band before the footer (a full-bleed hero-scale wordmark). Q1 audit confirmed this primitive doesn't exist in the design-system barrel — it was aspirational. With H3's modest wordmark living inside the footer's top zone, the standalone band becomes redundant. Recommendation: absorb / drop. The footer's wordmark + tagline is the only wordmark band on the page (besides the sticky header itself).

3 · Footer renders on every page (not landing-only like the MatchStrip)

By analogy with the MatchStrip (Q5 of Checkpoint C locked G3 — landing-only), one might ask whether the footer is also landing-only. It's not. Footer carries colofon (copyright, address, legal links, cookies) which the site needs on every page including detail pages, hero-less utility pages, and error pages. MatchStrip is a forward-looking module; footer is a legal/wayfinding anchor. Recommendation: render on every page via the root layout slot.

4 · Mobile collapse — 3 columns stack vertically; ink bar wraps to 2 rows

Below ~768px, the directory's grid-template-columns: 1fr 1fr 1fr collapses to 1fr (single column, columns stack: Ontdek → Doe mee → Bij de club). Each column keeps its mono-caps heading + bottom-rule + body-font links. Wordmark scales from 44px to 32px on mobile; tagline scales from 28px to 22px.
