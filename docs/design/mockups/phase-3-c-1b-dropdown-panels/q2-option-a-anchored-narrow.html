De club · meer dan een eeuw

Een club met een eigen plek

KCVV Elewijt speelt al meer dan honderd jaar in het amateurvoetbal. Onder de eik aan de Hollestraat is de zaterdagochtend een gewoonte.

Reads as: three identical mechanism-panels of different lengths. The panel is a list-of-routes; the data shape (4 / 10 / 11) sets only the height. No editorial layout, no per-panel character. Quietest of the four Q2 options — the surface (C ink-fill) does all the visual work, the layout does none.

Geometry

  • min-width: 244px
  • Anchored to trigger edge (left for left-half triggers, right for last)
  • top: calc(100% + 1px) — sits on the 1px ink seam
  • z-index: 60
  • Single column, em-dash bullet, 11px mono caps

Pros

  • One layout — no per-panel composition decisions to make in Q3.
  • Mechanically simple — same component shape across all 3 submenus.
  • Lowest implementation cost — no width branching, no responsive grid.
  • Predictable hover hit-area on every panel.
  • De club's 11 items at ~40px row = ~440px tall — fits 768px viewport easily.

Cons

  • Teams' 4 fixed tabs render as nondescript routes — loses the "tab" affordance entirely.
  • De club's 11 items become a tall column — readable but visually monotone.
  • No editorial differentiation between "section listing" and "route shortcut".
  • Q3 (Composition) effectively collapses to "what's the row vocabulary inside" — no per-panel layouts to drill.
