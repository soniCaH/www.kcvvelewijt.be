Reads as: a quietly highlighted line. The active item is unmistakable to anyone scanning, but doesn't shout. Same vocabulary as the top-level nav active state (jersey colour shift) — consistent with the masthead.

Pros

  • Smallest visual disturbance — the panel reads as a uniform list with one bright line.
  • Matches the top-level nav active state — vocabulary consistency.
  • No new structural element — single CSS rule swap.
  • aria-current="page" carries the semantic weight; visual is supportive, not load-bearing.

Cons

  • Could be missed by users with mild colour-vision differences — green-on-dark contrast carries the entire signal.
  • If the user's cursor is hovering another item, the hover state competes for attention with active.
  • No spatial / structural cue — relies entirely on colour.
  • Reads less like "you are here" and more like "this one is special" — slight semantic ambiguity.

WCAG check

  • jersey-bright #22c55e on ink-soft #1f1f1f: ~10:1 — passes AAA
  • aria-current="page" provides screen-reader semantics
  • Reduced motion / contrast preferences: no impact (no motion, contrast already high)
