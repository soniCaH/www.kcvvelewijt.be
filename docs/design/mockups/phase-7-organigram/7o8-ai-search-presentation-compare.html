Decision 7o8 — AI/semantic search presentation

B ★ aanbevolen  Antwoord-vooraan met échte CMS-tekst. Het serveert precies jouw zorg — wie een vraag stelt wil een antwoord, geen lijst — zonder het hallucinatie-risico van C, want de getoonde tekst is de clubeigen samenvatting van de top-match, niet die van het model. Gegate op de score-drempel die de backend al kent (≥ 0.5): sterke match → vooraan; zwakke match → terugval op de lijst (A). De LLM answer blijft uit de hub-dropdown (kandidaat voor een latere /zoeken-pagina).

  • Slim-signaal (alle varianten): een sparkle + "Slim gezocht / Beste match" mono-hint bovenaan de dropdown, zodat de gebruiker ziet dat natuurlijke taal werkte. Géén nieuwe merkkleur — jersey-deep + de bestaande iconografie.
  • Async: mensen (keyword) verschijnen meteen; de antwoord-lane toont een shimmer en vult zich als semantisch settelt. Faalt het endpoint → stille terugval op keyword (de PRD-vloer), zonder "AI"-hint.
  • "Meer resultaten / enter": de dropdown ís de resultatenset (geen aparte resultatenpagina in deze fase). Klikken op een rij of "lees volledig antwoord →" opent de accordion in de finder. We tonen onderaan géén nep "druk enter voor meer".
