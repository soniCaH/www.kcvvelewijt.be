Orbit the club.

The org as a real spatial structure you rotate, zoom and fly through — depth carries hierarchy, the whole thing turns. Maximum memorability; genuinely unique for a village club. Cost is real: three.js / react-three-fiber, a WebGL + perf budget, label legibility in 3-D, and a mandatory 2-D fallback for reduced-motion / low-power / screen-readers. Best treated as the ◎ enhancement on top of a solid 2-D base, not the only way in.

Decision 7o3 — the explorer interaction

A safe 2-D tree (cheap, low wow). B spotlight drill-down — the strongest pure fix for orientation + mobile (you’re never lost, never dragging). C radial constellation — the whole club as one figure, with a 3-D orbit toggle. D ◎ full 3-D spatial — maximum wow, heaviest build, needs a real 2-D fallback.

These compose. The responsible shape for your “3-D is awesome” + “don’t let me get lost / fix mobile” is a solid 2-D base (B’s spotlight nav) that the ◎ toggle lifts into 3-D (D’s scene) — progressive enhancement, with C’s radial as a candidate base if you want the whole-shape view. Pick the base + how far the 3-D toggle goes, and I’ll lock the combination (and flag 3-D as its own build phase in the PRD, since it’s the heavy part).
