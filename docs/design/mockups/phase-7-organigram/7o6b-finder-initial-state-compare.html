Decision 7o6b — the “Alles” landing at scale

A ★ capped preview — each category shows its top 3 + “Alle N →”; the category chip filters to that category’s full list. Browsable on load, caps the render no matter how big the corpus grows, and reuses the Structuur “Toon alle 23 →” pattern. B category accordion — Alles shows ~6 collapsed category rows; one click expands a category. Most compact load, but every question is two clicks deep. C search-led — no list until you type or pick a theme; leans hardest on search (which the hero already provides).

“Top 3” needs an order with no popularity data — use existing declaration/`sortOrder` or alphabetical (no fabricated “most asked”). Selecting a category chip → that category’s full accordion list. Search (hero or a finder-local field) always shortcuts straight to matching questions.
