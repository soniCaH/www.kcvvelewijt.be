Phase 4.5 · R10 · Three options visualised

The three options for handling R10, side by side.

Same 6 articles, three different card-structure treatments. Scroll to compare; the differences are entirely structural (frame nesting + image-vs-display content), the data is identical across all three.

Option 1 — Flush-edge only Option 2 — Flush-edge + display variants Option 3 — Hold (current state)
Option 1 · Minimal fix
Flush-edge structure, all cards have photos

Drop the inner ``. Image flushes to top + sides of the outer card. Ink rule divides image from meta panel. All 6 cards render with the article's `coverImage` as the top region — including event and transfer cards (which would use their own cover image, not a display block).

Trade-offs Matches source image's frame structure Single, predictable card primitive No new prop on NewsCard Event / transfer / column cards still need a coverImage No room for the display-text variants from the source image

Option 2 · Full source-design match
Flush-edge + per-type display variants

Same flush-edge structure as Option 1, PLUS new `displayMode` prop on `` that swaps the top region for display-text variants where it suits the article type: event cards show a date stamp, transfer cards show a display headline, columns can show a pull quote. Photos still default for interview / announcement / matchverslag.

Trade-offs Closest match to the source image Cards without strong photos still feel intentional Editorial flexibility per articleType New `displayMode` prop + 3 sub-templates inside NewsCard Editor decides what to fill in (date / headline / quote) per article More implementation surface area

Nested TapedFigure inside TapedCard (no change)
