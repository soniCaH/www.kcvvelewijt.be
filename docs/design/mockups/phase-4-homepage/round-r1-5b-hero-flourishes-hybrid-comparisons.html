Phase 4.5 · Round R1.5b · Hero per-articleType flourishes — HYBRID

Detail-hero compositions, ported · one signature overlay per variant where it earns it.

After comparing the R1.5 overlay-first proposals to the existing detail-page heroes (`InterviewHero`, `AnnouncementHero`, `EventHero`, `TransferHero`), this round adopts the port + one keeper approach: re-use the data-aware compositions the detail heroes already ship (joined names, ported kicker meta, directional kickerLabel, event ageGroup tag), render them in retro-fanzine vocabulary (TapedFigure, EditorialHeading), and keep ONE overlay artefact per variant only where it pays for itself visually.

Image sizing also bumps significantly — the prior round capped the taped polaroid at ~360px; these mockups go to ~440px (portrait) / ~520px (landscape) to match the photographic emphasis of the detail-page heroes.
