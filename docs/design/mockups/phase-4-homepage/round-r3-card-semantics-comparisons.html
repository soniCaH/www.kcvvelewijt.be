Phase 4.5 · Round R3 · Per-card semantics vs rhythm

Card backgrounds: type-driven semantic, or slot-driven rhythm?

With R2.B locked (3×2 equal news grid), the question is how each card's background is picked. Three strategies below. Each variant shows TWO distribution scenarios — a "balanced grid" (mixed article types) and a "transfer-heavy grid" (winter window with 4 transfers in a row) — so we can judge how the strategy holds up when content reality is uneven.

Calibration Transfers cluster seasonally: heavy in June–July (incoming) and January (winter). A balanced grid sees ~1 transfer per 6; a transfer-heavy grid sees 3–4. The "transfer-heavy" scenario is the stress test for any strategy that ties background to articleType.
R3.A — Slot rhythm (status quo) R3.B — Per-articleType (brief) R3.C — Hybrid
R3.A · Status quo (extended)
Slot rhythm — backgrounds cycle by position
The current Phase 4 pattern, extended to 6 slots: cream → jersey-deep → cream-soft → ink → cream → cream-soft. Background is decoupled from article type; variety is layout-driven, not content-driven. Visual texture is predictable regardless of which types fill the grid.
Trade-offs Predictable visual texture every grid Resilient to content clustering No data-driven re-renders Background carries no semantic signal Diverges from brief §6 'variant treatment per type is intentional'

R3.B · Brief §6 proposal
Per-articleType — background = type signal
Background is determined by `article.articleType`: transfer → jersey-deep, interview / announcement / event → cream. Background carries semantic meaning — visitor learns "green = transfer" over time. Pattern recognition for repeat visitors.
Trade-offs Matches brief §6 'variant treatment per type is intentional' Pattern recognition for repeat visitors Visually flat when grid is mono-type (5–6 cream) Visually heavy when transfer-heavy (4+ jersey-deep) No paper-stamp variety in calm periods

R3.C · Hybrid (Recommended)
Transfer always jersey-deep; others cycle
Transfers ALWAYS render on jersey-deep — preserves the semantic "green = transfer" signal from the brief. All other types fill the remaining slots with a 3-bg cycle (cream → cream-soft → ink → cream → cream-soft → ink), preserving rhythm variety. Best of both: semantic on transfer, rhythm on everything else.
Trade-offs Semantic signal preserved (transfers stand out) Paper-stamp variety on the 80%+ non-transfer slots Mostly resilient to clustering Slightly more complex assignment logic Transfer-heavy grids still get 3–4 jersey-deep tiles
