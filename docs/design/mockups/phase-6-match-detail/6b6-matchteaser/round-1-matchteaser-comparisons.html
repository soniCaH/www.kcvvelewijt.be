Phase 6.B · Drill round 1 · `<MatchTeaser>`

How should the teaser card look at default + compact?

Three structural shapes. Each variant renders the default + compact size side-by-side in the same panel so the relationship between the two sizes is visible. Same finished match across all variants — KCVV Elewijt 3 — 1 RC Mechelen, KCVV-home, played 14 June 14:30 at Sportpark Elewijt. Theme (light/dark), edge states + highlight treatment all deferred to round 2.

Pick one structural shape · round 2 covers theme + states
Variant A

Mini-hero.

d2's ticket-card shape at smaller scale. Left stub (~80px) = big display date + kickoff time. Right body = teams + score + competition mono row. Same dashed-divider chrome as the page hero (no perforations per d2 T2). Strongest vocabulary coherence with <MatchHero>.

Pros

  • Same component family as <MatchHero> — reads up + down the size scale as one vocabulary
  • Dashed-divider stub motif is distinctive at all sizes
  • Date stays prominent — readers scan slider/calendar rows by date

Cons

  • Stub width (~80px default, ~64px compact) eats horizontal space — fewer characters for team names
  • On dark theme (homepage slider): stub tint needs to invert/adapt — not free
  • Compact loses venue + competition — venue drop is unavoidable for any compact variant though
Variant B

Vertical stacked.

Date kicker on top, teams stacked vertically (home above away) with score on the right column. Venue / competition mono row at the bottom. Different shape from <MatchHero> — intentional, the teaser plays a different role (navigation, not page anchor).

Pros

  • Aspect ratio natural for the homepage slider — taller cards read as cards, not as rows
  • Highlighted team (home in this case) reads top-down — natural eye flow
  • Score-column stacking (3 over 1) mirrors the team rows; reads naturally
  • Dark theme is straightforward — invert text + bg, no special chrome adaptation

Cons

  • Most different from <MatchHero> — readers have to learn two card shapes for the same domain
  • Compact still has decent height; not the most space-efficient for calendar rows
  • Score-column stacking can read as "3 — 1" vertical OR as two separate digits — risk of misread on edge cases
Variant C

Horizontal fixture row.

Date column on the left | home (shield + name) | score | away (name + shield) | optional venue/competition strip below. Reads like a fixture-list row. Most space-efficient; compact is just a smaller padding + smaller font.
