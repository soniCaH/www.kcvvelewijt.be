Phase 6.B · Drill round 1 · Lineup + Events sections

How should lineup + events sit below the hero?

Three structural shapes for the body content that renders below <MatchHero> on finished matches. Both sections auto-hide for upcoming. Data is identical across all three: KCVV Elewijt 3 — 1 RC Mechelen, 7 events, 11 starters + 4 subs per team.

Pick one structural shape · per-row refinements in round 2
Variant A

Classic separate sections.

Two blocks. Opstellingen. first (home + away as two side-by-side columns), <StripedSeam>, then Wedstrijdverloop. as a single timeline ordered by minute. Conservative. Wraps existing <MatchLineup> + <MatchEvents> primitives 1:1.

Variant C

Events first, lineup collapsible.

Two blocks reordered. Doelpunten + kaarten. first (the part most readers want to see), then a collapsible Volledige opstelling. block at the bottom (default-collapsed via native <details>). Optimises for typical reading flow.

Pros

  • Matches typical reading flow: scoreboard story first, roster on demand
  • Saves significant vertical space (lineup is ~30 rows tall — hidden by default)
  • Zero new design-system primitive (native <details> is accessible by default + ships with all browsers)

  • Less "redesign-distinctive" — uses a convention familiar from any web form
