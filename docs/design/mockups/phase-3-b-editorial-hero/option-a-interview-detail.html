Variant interview — locked.

Detail-page hero for /nieuws/[slug] when articleType === "interview". Hero reuses the locked Asymmetric Broadsheet shell + announcement-locked slots. The variant- specific composition is a <SubjectsStrip> below the hero, parallel to the TransferFactStrip / EventFactStrip pattern. Each subject renders as a <PlayerFigure> polaroid (caption overridden for interview context).

★ Locked decisions

  • Q1 · Subject placement — Option 2: article.coverImage in hero right column · <SubjectsStrip> below the hero (same pattern as TransferFactStrip / EventFactStrip).
  • Q2 · N counts — N=1 (main): polaroid + pull-quote inside the strip · N=2: duo with italic display "&" · N=3: three polaroids inline, no separator · N=4: 2×2 grid.
  • Q2b · Mobile (N≥3) — strip flips to compact mono-caps list (display italic name + role + jersey/functionTitle/customRole). Polaroids do NOT render on mobile when N≥3. N=1 and N=2 keep polaroids on mobile.
  • Q3 · PlayerFigure reuse — same polaroid TapedCard shell as /spelers/[slug]. Caption slot overridden for interview context: display italic name + mono-caps role + jersey.
  • Q4 · Subject kinds — uniform polaroid treatment regardless of kind. Photo source switches per kind: player → psdImage · staff → photo · custom → customPhoto. Caption adapts (player: name + position + #jersey · staff: name + functionTitle · custom: customName + customRole). customPhoto required by existing schema validator — no schema change.
  • Pull-quote source (N=1) — first qaPair in body with tag in [key, quote]; attribution from respondentKey (or the only subject when N=1). Hides quote gracefully when no key/quote-tagged pair exists.
  • Press-down hover — none in this variant; no CTAs in the strip.
