★ 03 · RENDER RULE MATRIX — landing only · 2 states (hidden / upcoming)

Wanneer rendert de strip?

The strip is rendered by the landing layout slot, not the root layout. Detail-page templates inherit from a layout that omits the slot; the strip is therefore literally absent from those page subtrees in the DOM.

Page type Has upcoming match Strip render Reason
Homepage / Yes Render Landing surface — primary front door, next-match is highest-priority info
Homepage / No Hidden (null) Component returns null. No placeholder, no skeleton.
Section index — /nieuws Yes Render Landing layout slot
Section index — /evenementen · /teams · /jeugd · /sponsors Yes Render Same landing layout slot
Section indexes No Hidden null from data layer
Article detail /nieuws/[slug] (any articleType) Yes / No Hidden Detail-page template uses chrome-free layout
Single team page /ploegen/[slug] Yes / No Hidden Detail-page template
Single player page Yes / No Hidden Detail-page template
Match detail /wedstrijden/[id] Yes / No Hidden The match itself owns the page; strip would be redundant
Hero-less utility (/zoeken, /sitemap, errors) Yes / No Hidden Not landing surfaces. Header-only chrome.
