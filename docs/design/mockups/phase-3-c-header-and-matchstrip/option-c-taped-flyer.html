PHASE 3 · CHECKPOINT C · SITEHEADER + MATCHSTRIP · DIRECTION C ★ TAPED FLYER 2026-05-03

Een geplakt affiche.

Direction C reads the SiteHeader as a TapedCard pinned to the top of the page — cream paper, ink border, paper shadow, and a soft tan masking-tape strip at each upper corner. The MatchStrip beneath wears a TicketStub treatment: dashed-line rule between segments, perforated edge, ticket-style CTA. Most fanzine-leaning of the three checkpoint options, and the highest in primitive-reuse density — TapedCard, TapeStrip, and TicketStub all live in the chrome.

★ 01 · SITEHEADER — DESKTOP DEFAULT — TapedCard wrap · TapeStrip top corners · contents-list nav

Een kop als affiche.

The wordmark sits in Freight Display 900 with an italic accent on Elewijt in jersey-deep green. The nav follows as a printed contents-list row — leading section glyph § in italic serif, then mono caps separated by middots. A search affordance and a primary WORD LID button sit at the right rail. Tape strips are --color-tape rotated 5–6° outward.

↑ FOR

Header reads instantly as part of the redesign vocabulary — same paper foundation, same press idiom, same canon of corners and shadows. Tape strips reinforce the fanzine direction without polluting the nav with motion or rim-light. The contents-list nav (§ + middots) is a typographic move that costs zero tokens and zero new icons.

↓ AGAINST

A taped header sits "loose" on the page — visually it is a card pinned in place, not a chrome rail. For users expecting a flat opaque app-bar this can feel less stable. Tape strips also require rendering decoration above the card edge, which complicates strict overflow-clip patterns at very narrow widths.

★ 02 · SITEHEADER — MOBILE — wordmark + hamburger · tape strips kept

Op een klein formaat.

On mobile, the contents-list nav and primary CTA collapse behind a paper-card hamburger. The TapedCard wrapper and tape strips remain so the header still reads as a flyer pinned to the top — just narrower. Wordmark drops to 26px; tagline hides.

MOBILE < 768px
KCVV Elewijt

↑ FOR

Mobile collapse is a single wrapper rule — the TapedCard chrome itself does not change, only what it contains. Tape strips shrink proportionally (70px) keeping the flyer feel without crowding the wordmark. Hamburger is a paper-card mini, same press idiom as the desktop search button.

↓ AGAINST

Tape on a narrow card sits close to the wordmark — at very small widths (<320px) the left tape can visually crowd the K. Mitigation: a min-width clamp on the card or an even narrower tape. Drawer state itself is out of scope for this checkpoint.

MatchStrip wears a TicketStub aesthetic. Segments are split by a 1.5px dashed ink rule — same idiom as a paper match ticket. Four states: hidden placeholder (no upcoming fixture), upcoming (shield + tickets), live (LIVE pill + score), concluded (final score + match-report stub). The component never inflates the page — it sits as a flat sub-header band beneath SiteHeader.

↑ FOR

The dashed-rule treatment between segments is a one-token operation — border-right: 1.5px dashed ink. It evokes a torn ticket without needing actual SVG perforations, and the same rule scales cleanly to all four states. Live pill heart-beat is the only motion, and it is bound to a single element with aria-label="Live".

↓ AGAINST

Four segments at desktop widths is dense: shield + opponent + competition + CTA fits 1100px+ comfortably but compresses below 900px. Production needs a tablet rule that drops the competition segment first and keeps shield + CTA as the priority two. The hidden state intentionally renders empty space to prevent CLS — owners should confirm that's preferred over absent-from-DOM.

↑ FOR

Stacking the two as a single sticky assembly keeps total chrome height predictable and lets the page below scroll under one continuous shadow. Sharing the ink border between header and strip reduces visible rule count from two to one — quieter than the alternative.

A combined sticky block is taller (≈148px desktop). Pages with their own EditorialHero need to budget for that vertical space. If the strip is hidden (state A), the header alone sticks at ≈84px, which is fine — but transitions between states need to be opacity-only, never height-collapse, to avoid scroll-jank.

★ 05 · DIRECTION-LEVEL TRADEOFFS — summary across all four surfaces

Waarom deze richting.

Direction C is the most fanzine-leaning of the three. It pushes the chrome closer to editorial — the header reads as a flyer pinned to the page, the strip reads as a torn ticket. It also has the highest primitive-reuse density: TapedCard, TapeStrip, and TicketStub all appear in chrome that runs on every page.

↑ FOR

Maximum primitive reuse — TapedCard is the wrapper, TapeStrip the corner flourish, TicketStub the strip aesthetic. Every decision maps back to a primitive already locked in the catalogue. The fanzine reading lands harder than directions A or B without introducing a new shape vocabulary. Tape colour (--color-tape) is a single new token. Press idiom is canonical (translate(1px,1px) + shadow loss). Mobile collapse is structural, not a separate component.

↓ AGAINST

Highest editorial commitment of the three. Tape strips above the header card edge complicate strict overflow rules and can read as decoration where chrome usually reads as system. Two stacked chrome elements with their own decoration (tape + dashed rule) add visual weight at the very top of every page — pages with a loud EditorialHero need to defer to this assembly visually. Owners who want a quieter, more app-bar-like header should pick direction A or B; this is the maximalist option.

★ NOTES — token map, structure decisions, production handoff

Resolved

  • SiteHeader wraps in <TapedCard> — cream surface, border-2 ink, --shadow-paper-sm
  • Two <TapeStrip> at top-left (-6°) and top-right (+5°) — soft tan --color-tape + faint shadow, no rim-light
  • Wordmark: Freight Display 900, italic accent on "Elewijt" in --color-jersey-deep
  • Nav reads as a typographic contents list — leading § glyph in italic serif, mono caps separated by middot ·
  • Search button: 36×36 ink-bordered cream paper-card mini, typographic ⌕ glyph (no Lucide)
  • WORD LID: ink fill, cream label, jersey-deep paper shadow, canonical press idiom (translate(1px,1px) + shadow loss)
  • Mobile collapse: same TapedCard wrap; nav + CTA replaced by single hamburger paper-card mini; tape strips shrink to 70px
  • MatchStrip wears TicketStub flavour: cream-soft surface, dashed-rule inter-segment dividers (1.5px dashed ink), TapedCard's bottom border becomes the strip's top border (shared rule)
  • 4 states: hidden placeholder (dimmed, italic copy), upcoming (shield + opponent + competition + Tickets CTA), live (LIVE pill + running score + heart-beat), concluded (final score + Verslag CTA)
  • LIVE pill animates a single 7×7 dot via opacity only — no glow, no rim-light, respects reduced-motion
  • Stenciletta off-limits — wordmark uses Freight Display, kickers use IBM Plex Mono
  • Combined header + strip becomes position: sticky; top: 0 in production; transitions between strip states are opacity-only, never height-collapse

Comparison to siblings A and B

Direction A — flat opaque header, no tape, rectangular MatchStrip with solid inter-segment rules. Quietest. Reads as a system app-bar.

Direction B — TapedCard header (no tape strips), TicketStub strip (no perforation, only dashed rules). Mid-fanzine. Reads as paper but not as decoration.

Direction C (this) — TapedCard + TapeStrip + TicketStub together. Loudest. Reads as a flyer pinned to the page with a torn ticket beneath. Highest primitive-reuse density of the three.

Motion budget

  • Tape strips: static — no entry animation, no hover transform. Tape is decoration that pretends to be physical; movement breaks the illusion.
  • Search button, hamburger, WORD LID, strip CTAs: canonical press idiom — translate(1px,1px) + shadow loss, 300ms ease. No additional hover effect (no colour swap, no border thickening).
  • LIVE pill dot: opacity 1 ↔ 0.35, 1.4s ease-in-out infinite. Single property (opacity) keeps the animation cheap and respects reduced-motion when scoped.
  • Nav links: hover swaps colour to --color-jersey-deep only — no underline animation, no transform.
  • Sticky behaviour: native position: sticky. No scroll-triggered shadow change, no colour fade — the assembly looks identical at scrollY: 0 and scrollY: 1000.
  • State transitions in the strip: opacity-only fade 200ms when shifting between A → B / B → C / C → D. Never animate height.

Edge cases handled in this mockup

  • Long opponent names — match-strip__team-name uses line-height: 1.1; production should add max-width + ellipsis at small widths
  • Kickoff time without minutes — date format ZAT 09 MEI · 15:00 always renders the time even when it's 00:00; production should drop the time when source data has time component 00:00 (per PSD memory: date field carries dummy 00:00 from PSD /games endpoint)
  • Match identification — never compare by club name; always by teamId. The KCVV / opponent shield assignment in production reads homeTeamId === selectedTeamId to determine "we are home" (per PSD memory)
  • Sponsor logos / shields with no asset — fall back to mono-caps initial in the shield slot, never render a broken <img>
  • State A placeholder copy is editor-tunable via Sanity setting; default Dutch copy shown here
  • Reduced-motion: when prefers-reduced-motion: reduce is set, the LIVE pulse is replaced by a static dot at full opacity and all 300ms press transitions drop to 0ms
  • RTL — wordmark, nav, CTA group flip via flex-direction: row-reverse on the header; tape rotations mirror (+6deg / -5deg); dashed inter-segment rules are direction-agnostic
  • Print stylesheet — tape strips and shadows hide; dashed rules become solid; nav becomes a flat list; LIVE pulse stops
  • High-contrast mode — relies on system border-2 ink and ink-on-cream text only; tape strips remain visible at full contrast (no transparency-based effects)
