PHASE 3 · CHECKPOINT C · DIRECTION B ★ INK BAND 2026-05-03

Een concourse in ink.

The site header is a single solid ink band — cream wordmark, cream mono caps nav, jersey-green WORD LID that pops off the dark like an LED. Underneath it sits a quieter cream MatchStrip with a 6px jersey-green rail down its left edge. Two bands, hard contrast, club-shop signage feel — reads like a stadium concourse sign welcoming you into the page.

★ 01 · SITEHEADER · DESKTOP DEFAULT — ink band, cream wordmark, cream mono caps nav, jersey-green CTA

Het signage-band.

Wordmark sits hard left in Playfair italic 900, cream on ink. Nav links push to the right margin in IBM Plex Mono caps with 0.12em tracking — every label reads as a stamped sign, not a soft web link. Active state underlines in jersey green. The search affordance is a hairline cream square with a typographic magnifier glyph; WORD LID sits on the far right as the only saturated colour on the band.

↑ FOR

Maximum contrast against the cream paper foundation — the header acts as a literal frame for the page, the way a kiosk fascia frames a concourse. Cream-on-ink mono caps nav reads as inarguably "club sign", not as a generic SaaS top bar. The jersey-green CTA is the only colour on the band, so it cannot fail to draw the eye. Search and nav remain quiet so they don't compete with content below.

↓ AGAINST

A solid ink band is heavy — it visually weighs more than the cream-on-cream alternative and crops the page from above. Light-mode reading is unaffected, but pages with an ink hero just below (e.g. EditorialHero on dark ground) will need a clear seam treatment so the header doesn't merge with the hero. Mitigation: MatchStrip in cream provides natural breath between the two ink surfaces.

★ 02 · SITEHEADER · MOBILE — hamburger + collapsed nav, wordmark scales, CTA hides

Het compacte band.

On narrow viewports the nav, search, and primary CTA all collapse behind a single hamburger affordance — a hairline cream square mirroring the search button on desktop. Wordmark drops to 22px so it doesn't crowd the chrome; the secondary "SINDS 1942" caption remains because it carries visual identity at a glance, and at this scale it costs almost nothing.

↑ FOR

One band, four payloads, identical chrome. Cream bg + jersey rail gives MatchStrip a distinct identity from the ink SiteHeader without introducing a third surface colour. Ink-bordered shields scale to both PSD club codes (no logo asset required) and full club logos once available. The 6px jersey rail is the connective tissue — same colour as the WORD LID button above, so the eye reads "this band belongs to KCVV's chrome system, not to a third-party widget".

↓ AGAINST

The band crowds quickly — kicker + teams + meta + CTA + sep rules is a lot to fit in 56px of vertical real estate, and at narrower viewports something must drop. Likely casualty in tablet view: the kicker label collapses; in mobile view: the meta date string collapses to a tap-through. Live-state's red pill against the cream + jersey rail also flirts with three saturated colours in a very small band — must be tested for accessibility contrast at the rail/pill boundary.

★ 03b · MATCHSTRIP · MOBILE COLLAPSE — kicker drops, meta truncates, CTA condenses to glyph + label

Op een klein scherm.

The 6px jersey rail stays — it is the band's identity.

The jersey rail + ink-bordered shield + score type carry the identity at any width. Even at 360px the strip never crosses two lines — a hard visual rule that keeps the chrome from eating the viewport. ★ NOTES — resolved decisions, open questions, token map

Resolved

  • SiteHeader is a solid --color-ink band — cream wordmark + cream mono caps nav + jersey-green CTA
  • Wordmark uses var(--font-display) italic 900 with a small mono caption beneath; caption hides under 560px width
  • Nav links are mono caps (IBM Plex Mono, 12px, 600, 0.12em tracking); active state underlined in jersey-green 2px
  • Search affordance is a typographic glyph (⌕) inside a hairline cream-bordered square — never Lucide
  • WORD LID button uses the press-down hover canon: box-shadow → none + translate(1px, 1px)
  • MatchStrip sits directly under SiteHeader with no margin gap; it has 2px ink bottom border + 6px jersey-green left rail
  • MatchStrip renders four states — upcoming, live, concluded, no-upcoming (the last is a Storybook reference; in production the band is unmounted entirely)
  • Opponent shields are StampBadge instances: 32×32, 2px ink border, 2-letter code in mono caps; KCVV home shield gets jersey fill
  • LIVE pill is the only saturated red on the page — pulsing dot is the only permitted motion in this chrome assembly
  • Combined sticky chrome is 110–120px on desktop, 96–104px on mobile (single-row header + slimmed strip)
  • No glows, no rotations, no border-radius, no Stenciletta — every choice maps back to an existing primitive (see reuse table above)

Open

Once locked, ship SiteHeader + MatchStrip as Phase 3 components in apps/web/src/components/layout/, retire the existing PageHeader, and add Storybook stories per state.
