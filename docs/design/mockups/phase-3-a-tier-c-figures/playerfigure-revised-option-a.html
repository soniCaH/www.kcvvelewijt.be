PHASE 3 · CHECKPOINT A · DIRECTION A ★ PHOTOBOOTH STRIP — REVISED 2026-05-03

Twee aparte staten.

The original <PlayerFigure> spec mixed a circular photo face with a drawn jersey + arms — read as "Mickey Mouse" and was rejected. This revision splits the component into two mutually-exclusive runtime states: a photo state (rectangular psdImage + editorial chrome only — no drawn body parts) and an illustration state (a complete flat silhouette with jersey, arms, and stylised head — no embedded photo). One or the other, never both.

★ 01 · PHOTO STATE — rectangular psdImage on TapedCard with TapeStrip + cream frame

Wanneer er een foto is.

psdImage exists for ~90% of squad players. The figure renders as a polaroid-strip rectangular crop, mounted in a cream-paper inner frame on a tilted TapedCard, with a TapeStrip across the top. Outside the card, the TicketStub stamnr, a MonoLabel for team + position, and the editorial name composition do all the talking. No drawn jersey. No drawn arms.

↑ FOR

Photo speaks for itself — rectangular crop respects the natural shape of psdImage (90% of squad). Cream inner frame + outer TapedCard + TapeStrip read editorial without dressing the player up. No drawn body parts means the photo is never competing with a graphic costume. Composition cleanly separates identity (photo) from metadata (ticket + label + name).

↓ AGAINST

Players without a usable psdImage need a fully different render path (Section 02) — the chrome is shared but the centre swaps. Tilt + tape strip means the photo is never perfectly upright, which reads "scrapbook" not "official roster shot"; some clubs prefer the latter. Mitigation: keep tilt under ~2° so it never feels rotational.

★ 02 · ILLUSTRATION STATE — full drawn silhouette, no photo well

Wanneer er geen foto is.

The fallback is a complete drawn figure — block-color silhouette in --color-jersey-deep on --color-cream-soft, showing jersey body with a vertical stripe and squad number, both arms, and an oval stylised head (no facial features — kept abstract so the figure stands in for any player). The drawing IS the figure. The same outer TapedCard chrome and same metadata column compose around it — only the centre swaps.

The illustration fallback is canonical — identical across all photo-state directions. Direction A/B/C governs only the photo state (above).

↑ FOR

The drawing is internally consistent — jersey, arms, and head all rendered as a single silhouette in one ink-line idiom. No attempt to fake a face, so no uncanny mismatch with a missing photo. Block-colour read holds at every size and prints flat. Squad number is part of the drawing, not pasted-on chrome — keeps the identity readable without a portrait.

The figure is generic by design — every photo-missing player shares the same silhouette. For coaches and staff this is fine; for first-team starters with no photo it can read as a placeholder rather than a portrait. Mitigation: editorial copy confirms it is a fallback ("NO PHOTO ON FILE"), and PSD sync is the long-term remedy.

★ 03 · PHOTO STATE — TIGHT CROP — same photo, chest-up crop, sharper tilt for hero / featured spots

Een scherpere uitsnede.

Same photo, tighter object-position — chest-up rather than full upper body. Pairs with a sharper card tilt for hero placements (player-of-the-match, profile page lead, sponsor handshake spotlight). Same chrome, same composition, only the framing changes.

↑ FOR

Both states share 9 of 10 chrome rows — only the centre slot differs. That makes the runtime branch a single conditional in the React component (psdImage ? <Photo /> : <Silhouette />), with all surrounding primitives composed identically. No new tokens, no new shapes, no new shadows.

↓ AGAINST

The illustration state introduces one new asset: the SVG silhouette. It must live in the design system as a single source-of-truth (proposed: PlayerSilhouette in packages/design-system/PlayerFigure/). One asset, but it is bespoke — not a composition of existing primitives.

★ NOTES — correction summary, runtime split, handoff

Correction

The original master-design PlayerFigure (circular photo face inside a drawn jersey + arms) was rejected for reading as "Mickey Mouse". This revision replaces that hybrid with two mutually-exclusive runtime states. Photo and drawn body parts are never combined in the same composition.

Runtime branch

Pseudocode: player.psdImage ? <PlayerFigurePhoto crop="roster|portrait" /> : <PlayerFigureSilhouette />. Surrounding metadata (TicketStub, MonoLabel, EditorialHeading, quote) is composed by the parent — independent of which state the centre renders.

Open

Owner pick between this revised Direction A and any sibling revised directions authored alongside. Once locked, ship the new <PlayerFigure> component as two sub-components (PlayerFigurePhoto, PlayerFigureSilhouette) plus a thin selector wrapper. Storybook stories per state + crop variant. Seed staging with at least one photo-missing player to exercise the silhouette path end-to-end.
