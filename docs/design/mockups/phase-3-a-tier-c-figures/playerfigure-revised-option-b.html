PHASE 3 · CHECKPOINT A · DIRECTION B ★ STAMPED BLOCK-PRINT PLAYERFIGURE (REVISED) 2026-05-03

Direction B — Stamped Block-Print · PlayerFigure (revised) · Phase 3 · Checkpoint A

PlayerFigure now lives in two mutually exclusive states. A photo state shows a rectangular psdImage desaturated and wrapped in a misregistered ink outline + postmark chrome — no drawn arms, no drawn jersey. An illustration state renders a full 2-tone block-print figure (ink + jersey-deep, deliberately off-register) — no photo anywhere. The two never combine.

▲

DESIGN CORRECTION

The original master-design hybrid (a photo face inside a drawn jersey with drawn arms) is rejected — it reads "Mickey Mouse". Misregistration here is between the photo and an OUTLINE rectangle (chrome), never between a photo and a drawn body. Photo and illustration are two separate states of the same primitive.

★ 01 · PHOTO STATE — rectangular psdImage · desaturated · misregistered ink outline · stamp chrome

Een foto, niet getekend.

The photo state is a single rectangular psdImage with editorial framing only: filter: grayscale(0.6) contrast(1.1), a 2.5px ink outline rectangle offset 3px down-right (the printed-fanzine "off-register" feel), and an optional jersey-deep hairline shifted the opposite way. There are no drawn arms, no drawn jersey, no drawn collar — the photograph carries the figure entirely.

↑ FOR

Reads as a documentary photograph in a fanzine — desaturated, framed by a misregistered printed rectangle. The misregistration is purely chrome (an outline shifted from the photo edge), so the photo itself is never violated by drawn anatomy. Works for ~90% of squad players whose only asset is a rectangular psdImage. Composes with locked primitives: TapedCard, StampBadge, TicketStub, MonoLabel.

↓ AGAINST

Two stacked outlines (ink + jersey-deep hairline) plus the TapedCard border plus the photo's own border = four edges around one image. At very small thumbnail sizes (<160px) the off-register hairlines compress to noise; the jersey-deep alt outline is the first to drop. Photographic quality varies wildly across the squad — the desaturation filter helps but cannot rescue badly-lit psdImages.

★ 02 · ILLUSTRATION STATE — full 2-tone block-print figure · ink + jersey-deep · 2.5px misregistration

Een tekening, geen foto.

The illustration state is a full drawn figure — head, torso, arms, jersey — rendered as a 2-tone block-print. The jersey-deep underprint sits 2.5px off-axis from the ink overprint via mix-blend-mode: multiply, producing the deliberate misregistration of a hand-pulled lino. There is no photo well, no photo placeholder, no rectangular crop anywhere in this state.

The illustration fallback is canonical — identical across all photo-state directions. Direction A/B/C governs only the photo state (above).

↑ FOR

First-class fallback for the ~10% of squad members without a usable psdImage, and an editorial choice for hero contexts that want a graphic rather than a snapshot. Two-tone ink + jersey-deep stays inside the locked colour token map. The misregistration is the same technique as the photo state's outline — visually consistent with the whole direction. No photo means no licensing or quality risk.

↓ AGAINST

The illustration is a fixed silhouette family — every figure looks roughly the same body and head. Per-player variation costs an illustrator pass (Phase 4 deliverable). Inline SVG ships ~1.5KB per figure; on a 25-player squad page that's ~38KB of vector. Mitigation: extract the silhouette to a single sprite and reference by <use>.

★ 03 · PHOTO STATE — CROPPED TIGHT — same desaturated photo · chest-up crop · same misregistered outline + stamp chrome

Een strakker kader.

For PlayerHero and profile contexts the photo state ships a tighter chest-up crop. Same desaturated psdImage, same misregistered ink outline, same postmark chrome — only the object-position shifts (center 18%) so the subject's face occupies the upper third of the rectangle. No additional drawn anatomy is introduced.

Same component, no new variant of chrome — the only delta is a CSS object-position token. Hero contexts get a face-led composition without breaking the catalogue's silhouette pattern. The misregistered outline frame still reads at hero scale because the offset is absolute (3px), not a percentage of the photo size.

↓ AGAINST

Tight crop depends on consistent face placement in source psdImages. Players photographed at different distances (some chest-up, some full-body) will land their face at different vertical positions under a fixed object-position: center 18%. Mitigation: an editor-tunable focal-point token in Sanity per-player asset.

↑ FOR

Net-new artwork is bounded: one SVG silhouette family (illustration state) and one CSS recipe (off-register outline rectangle, photo state). Every other surface is a locked primitive. The two states share their entire outer chrome — TapedCard, StampBadge, caption, MonoLabel, TicketStubs, EditorialHeading — so swapping between them at runtime is a single content prop, not a layout shift.

↓ AGAINST

Two states means two visual languages on the same squad page if some players ship photos and others ship illustrations. Mitigation: prefer photo state when a usable psdImage exists; reserve illustration state for staff/coaches and players who explicitly opt out. Document the selection rule in the component contract.

★ NOTES — direction-level summary, production handoff

Resolved

  • Photo state and illustration state are mutually exclusive. A PlayerFigure renders one or the other — never both at once.
  • Photo state = desaturated rectangular psdImage + 2.5px misregistered ink outline (3px down-right offset) + optional jersey-deep hairline (-2px, -2px). No drawn arms, no drawn jersey, no drawn collar.
  • Illustration state = full 2-tone block-print figure (head + torso + arms + jersey + shorts). Jersey-deep underprint at translate(-2.5px, 2.5px) + mix-blend-mode: multiply. No photo well anywhere.
  • The cropped-tight variant is the photo state with object-position: center 18% — same primitive, no new chrome.
  • Misregistration in the photo state is between the photo and an OUTLINE rectangle (chrome), not between the photo and a drawn body — this is the corrected spec.
  • Outer chrome (TapedCard, StampBadge, caption strip, MonoLabel, TicketStubs, EditorialHeading) is identical across both states.
  • Press-down hover canon (hover:shadow-none + translate(4px, 4px) + transition-all 300ms) applies to the staged TapedCard wrapper.
  • No radial glows, no rim-lights, no Stenciletta. Block-print is print-flat.
