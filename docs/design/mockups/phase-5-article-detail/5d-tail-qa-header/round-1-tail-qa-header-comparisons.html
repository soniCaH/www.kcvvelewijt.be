Tail Q&A section header — typography drill

Owner feedback from the Phase 5 grilling session (2026-05-20): the tail-grouped Q&A's centered <MonoLabel tone="ink">Q&A</MonoLabel> reads too quietly between <EndMark> above and the first <QaPair> below — "should be other typography, bigger". Drill resolves which typography variant becomes the locked header for the tail section.

Each option renders the same context — closing <EndMark> from the in-flow body, header treatment, first tail <QaPair> — at --container-prose width (680px) on cream, so rhythm and scale are visible at a glance. The locked Phase 3-b Q&A row shape is reused as the after-header context (monogram avatar · mono speaker tag · italic display-sm question · body-md answer).

Every variant composes only from already-locked primitives — no net-new vocabulary. See compare.md in this folder for the per-option source-primitive map.

A · kicker + heading

Kicker + display-md heading

Small <MonoLabelRow> "* Q&A *" kicker above a centered italic <EditorialHeading size="display-md">. Echoes the InterviewHero opening pattern ("* AFSCHEID DUBBEL *") and the EndMark "* Einde gesprek *" rhythm above. Quietest of the four; voice-driven, no rules, no seam.

Source primitives • <MonoLabelRow> (kicker)
• <EditorialHeading size="display-md"> italic
• Jersey-deep period (existing EditorialHeading convention)
Vocabulary delta None. Direct reuse of two locked primitives.

B · heading + rules

Display-lg heading + flanking rules

Italic centered <EditorialHeading size="display-lg"> flanked by 1px ink rules at 0.55 opacity. Same vocabulary as the locked Phase 3-b body section breaks (E0 in 5.d3), scaled up from display-sm to display-lg so it reads as a NEW section rather than a subsection. Strong consistency with the body's own break treatment.

Source primitives • <EditorialHeading size="display-lg"> italic

• Jersey-deep period (existing convention)
Vocabulary delta None. Same shape as <QASectionDivider title> at a larger heading size. Could land as a new prop on <QASectionDivider> (e.g. headingSize="lg") or compose directly.

C · seam + heading

StripedSeam + display-lg heading

Full-bleed <StripedSeam height="sm" colorPair="ink-cream"> above a centered italic <EditorialHeading size="display-lg"> with a small mono kicker beneath the seam. The seam is the canonical primitive for separating page regions; using it inside a single article reads as a register shift — "the page just turned, here's the bonus track". Highest visual energy.
