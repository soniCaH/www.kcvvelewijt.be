Trade-offs
  • Mobile 1 (collapse to prose) — simplest CSS, but the editor's "wide" intent disappears on phones. Anyone authoring "wide" implicitly means "let it stand out" — Mobile 1 silently downgrades.
  • Mobile 2 (collapse to bleed) — preserves dramatic intent. Two modes on mobile (framed prose vs. 100vw) — easy to test, easy to author. Possibly aggressive: a three-position desktop control collapses to two on mobile.
  • Mobile 3 (wide-ish 92vw) — preserves all three modes on mobile. A 4vw gutter visually distinguishes wide from bleed. Most fidelity to the editor's intent; slightly more CSS / more VR cases.
Constants across all options
  • prose on mobile = viewport minus 18–28px body padding (already responsive — no change from drill 5.A.1).
  • bleed on mobile = 100vw, tape suppressed (same as desktop).
  • Breakpoint = ~640px (existing md Tailwind breakpoint).
