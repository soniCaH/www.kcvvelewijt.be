articleImage — caption + credit (full / caption-only / PT-caption / none)

Drill 3.1 sub-round 2. Locks whether captions and credits become first-class schema fields, and how they render below the flat TapedFigure from Round 1.

Current schema (packages/sanity-schemas/src/articleImage.ts) has only image + alt + fullBleed. TapedFigure supports caption?: string + credit?: string — used today by NewsCard / EditorialHero / FeaturedEventBand. Three of the four options below propose a schema migration to wire the primitive's existing props.

Each option renders the same fixture: photo + a caption ("Trainer Vermeulen geeft de U13 instructies vóór de eerste set partijen.") + a credit ("Foto: An Verheyden") embedded in body prose.

Option A · caption + credit (string + string)
Matches TapedFigure's existing API 1:1
Adds two optional string fields to articleImage: caption and credit. Renders inside TapedFigure's existing <figcaption> — caption italic serif body-sm on the left, credit mono caps ink-muted on the right. Each row drops independently when its field is blank.
Schema migration ✓ add caption: string (optional)
✓ add credit: string (optional)
✓ TapedFigure's existing figcaption API

Option B · caption only (string)
One field, italic serif below the figure
Adds only caption: string (optional). No credit field at the schema level — image credit can be folded into the caption text if the author wants to attribute. Simplest migration; weakest provenance signal.
Schema migration ✓ add caption: string (optional)
✗ no credit field
⚠ credit must be appended manually inside caption text

Option C · PortableText caption + credit string
Captions can carry inline links + accent decorator
caption is a constrained Portable Text array (paragraph block + link + accent decorator). credit stays a plain string. Supports captions like "Lars Janssens, portretreeks", but requires TapedFigure to accept a PT renderer for caption (small primitive change).
Schema migration ✓ add caption: PortableText[] (constrained marks)
✓ add credit: string (optional)
⚠ TapedFigure caption slot needs PT support (small refactor)

Option D · No schema change (caption stays inline)
Author writes a paragraph after the figure
Schema stays as it is. Captions live as regular paragraph PT blocks after the image. The renderer detects the pattern (image followed by italic paragraph?) and styles the paragraph as a caption — OR the author manually flags "italic" on the paragraph. Most editor-flexible; weakest structural binding.
Schema migration ✗ no schema change
⚠ caption is a separate PT block, not structurally bound
⚠ editor must remember to italic-mark + style as caption
