InterviewCredits block layout — list / centered / sidebar / inline

Drill 5.d-int Round 2 of 2. Round 1 locked Q&A row marker to D (no number). This round decides the closing <InterviewCredits> block — placed after <EndMark> and before <VerderLezenRow>.

Each option renders the same 4-line credits sample at the natural article-end position. Typography is identical across options (mono caps key + italic display name); only the arrangement differs.

Rows of "KEY · Name" stacked, prose width

Option D · Inline with EndMark
Single italic credit line + closing star — tightest composition
EndMark and credits merge. One italic line carries author / interviewees / photographer / date as a comma-separated run; the star sits centered beneath. Like a film closing card. Lowest visual weight, most cinematic.
Vocabulary ✓ EndMark + inline italic line
Δ Each credit field becomes inline-run instead of "KEY · Name" row
Δ Long credit lists (5+ interviewees) wrap and read messy
Δ Loses key/value contrast — readers have to parse "Tom De Smet · Tekst" as a unit
