videoBlock — frame (TapedFigure with tape / TapedFigure no tape / plain bordered / no frame)

Drill 4.1. Locks the frame around the video poster/player on the cream surface. The locked <ArticleImage> Round 1 used a flat TapedFigure with a single tape strip — this round asks whether video should match, soften the decoration, or drop the frame entirely.

Each option renders the same fixture: 16:9 video poster (with a play-glyph overlay placeholder — the actual play affordance is drilled in Round 2), caption string from the videoBlock schema, embedded in body prose. poster is shown in every option so the comparison is honest — when no poster is set, the video element starts black and behaves identically across all four.

Consistency signal
  • Option A is the only one that achieves photo/video parity in the body — readers see one "framed media" family across both block types.
  • Options B–D each break parity in a different way: B keeps the frame but drops the decoration; C swaps the frame; D drops the frame entirely.
Out of scope this round
  • Play affordance (drilled in Round 2). The play glyph in every option is a placeholder; the actual button design is decided separately.
  • Width behavior (drilled in Round 3). Every option above is shown at --container-prose; the width enum decision is identical to the articleImage R3 decision and may inherit it.
  • Embed iframe rendering (YouTube/Vimeo) — same frame applies to both upload and embed paths. The play affordance differs (YT/Vimeo render their own; uploads render ours).
