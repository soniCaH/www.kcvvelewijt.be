In-body eventFact — multi-card stacking (identical / subtle alt / stronger alt / flatten-after-first)

Locked Round 1 · C (Taped polaroid). Round 2 decides what happens when 3 consecutive eventFact polaroids stack in a single article body — e.g. a tournament weekend with three days each as its own eventFact, or an announcement listing three kennismakingsmomenten back to back.

Same fixture across all four — three eventFacts: Steakfestijn (13 juni), Open trainingsmoment (20 juni), Paastornooi (4 juli). Each renders with the locked Round 1 polaroid card; only the stacking rule changes.

Option A · Identical rotation
Every polaroid rotates -0.4°
Every card uses the same transform. Predictable, stable. Reads as "three event cards" rather than "three pinned polaroids." Cleanest CSS (single rule); weakest fanzine character.
Vocabulary ✓ identical transform: rotate(-0.4deg)
✓ no per-card variation

Trade-offs
  • A · Identical — simplest, most predictable. Polaroid rhythm fades on repeat; reads as "the design language stalled".
  • B · Subtle alt — gentle variety. Each card feels hand-pinned without becoming busy. CSS-only (nth-of-type). Recommended path of least new vocabulary.
  • C · Stronger alt + tape variation — boldest fanzine register. Adds a second tape color (mint) to the vocabulary — used nowhere else yet. Could overpower body context.
  • D · First polaroid, rest flatten — explicit "featured + list" hierarchy. Solves decoration accumulation cleanly, but introduces a second visual variant for the same schema block (more code, more tests).
Reset rule
  • For Options A / B / C: rotation pattern resets every time a non-eventFact block (paragraph, image, qaBlock) sits between two eventFacts — the CSS sibling-selector approach (nth-of-type) handles this naturally inside one PortableText render.
  • For Option D: the "first polaroid" rule resets after any non-eventFact block too. A flat two-line eventFact only renders when it immediately follows another eventFact in the PT array.
