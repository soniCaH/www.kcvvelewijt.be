In-body eventFact — card vocabulary (mini-detail / ticket-stub / polaroid / two-line)

Drill 2.1 sub-round 1. Locks the visual for an eventFact PT block when it sits inline inside <ArticleBody>. Two cases share this visual: (1) inline-in-announcement (one event callout inside an announcement) and (2) multi-eventFact-in-event (subsequent eventFacts after the first one is absorbed by <EventDetailBlock>).

Every option renders the same fixture: Steakfestijn 2026, 13 juni, locatie + adres + capaciteit + competitionTag, ticket CTA. Wrapped in body prose paragraphs to show the rhythm inside an article. Fusing behaviour (consecutive cards) is held for Round 2.

Option A · Mini EventDetailBlock
TapedCard family, compressed (no sessions / no note)
Same vocabulary as the locked <EventDetailBlock>: cream-soft TapedCard, jersey tape strip top, 76px date-block left, tag pill + italic display title right, dotted ink-muted divider, meta <dl> below, CTA at the bottom. Compressed by dropping the sessions block and the note section (those stay exclusive to the page-end EventDetailBlock).
Vocabulary ✓ TapedCard + jersey tape (matches EventDetailBlock)
✓ 76px date block (same as EventDetailBlock head)
✓ italic display-sm title + dotted divider
✓ mono caps meta dl
✗ no sessions, no note (compression rules)

Date stub (jersey-deep) left, body center, CTA cell right

Family relation to <EventDetailBlock>
  • A · Mini-detail directly mirrors the locked EventDetailBlock — same tape, same date block, same dotted divider, same dl. Reader recognises "this is the same event object as the one at the article tail."
  • B · Ticket-stub uses the TicketStub primitive (already approved in design vocabulary). Strongest visual difference from EventDetailBlock — easier to tell "primary" from "secondary" event in a multi-eventFact event article.
  • C · Polaroid introduces a warmer ochre tape and a rotation — most decorative but adds the most new vocabulary.
  • D · Two-line strip has the lightest footprint, but weakest wayfinding when 3+ eventFacts stack consecutively.
Fields rendered (all options)
  • title (required), date, startTime/endTime, location, address, ageGroup/competitionTag (pill), ticketUrl + ticketLabel (CTA), capacity.
  • NOT rendered: sessions[] (exclusive to EventDetailBlock), note PT (exclusive to EventDetailBlock). These compression rules keep the inline card visually lighter than the page-end one.
