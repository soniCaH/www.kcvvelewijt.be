Phase 5 · Drill 5.d-table · Round 1 · #1840

htmlTable — cream-surface restyle (jersey-band / newsprint / dark card / minimal)

Drill 5.2. The legacy HtmlTableBlock wraps sanitized HTML in a scroll container with a gray header band, alternating white/gray rows, and a sticky first column on scroll. None of those colours sit on cream. Four cream-friendly restyles below.

Same fixture across all options: a 5-row schedule table (datum / tegenstander / locatie / scheidsrechter / uitslag). Wrapped in body prose. The horizontal-scroll affordance + sticky first column behaviour are preserved in every option (CSS omitted for brevity, identical to legacy).

Option A · Jersey-band header + alternating rows
Jersey-deep header band, cream/cream-soft zebra, 1px ink borders
Closest to a "club programme" table. Jersey-deep header band (cream text, mono caps), 1px ink borders on every cell, alternating cream + cream-soft body rows. First column bolded for row identification. Most "stadium signage" register.
Vocabulary ✓ jersey-deep header band
✓ MonoLabel header typography
✓ 1px ink-muted body borders
✓ alternating cream / cream-soft zebra

Option B · Newsprint hairlines
2px ink top + bottom, 1px dotted row dividers, italic display first column
"Sports section of a newspaper" register. No cell borders; 2px ink top + bottom bracket the whole table; rows separated by dotted ink-muted (Phase 3-b primitive). First column in italic display Freight — gives the row labels editorial weight. No alternating row colours; relies on typography for scannability.

Preserved behaviour (all options)
  • Horizontal scroll wrapper for tables wider than the prose column (legacy useScrollHint).
  • Sticky first column on scroll (legacy behaviour).
  • HTML sanitization rules unchanged (TABLE_SANITIZE_OPTIONS).
  • Editor authors raw HTML in the schema; no migration; styling is pure CSS / Tailwind on the wrapper.
Trade-offs
  • A · Jersey-band — strongest brand; jersey-deep header reads as "official club data."
  • B · Newsprint hairlines — quietest brand presence; strongest editorial body fit; relies on typography alone.
  • C · Dark card — best for dense numeric data (rankings, statistics); mono body aligns columns visually.
  • D · Minimal — lightest footprint; weakest column separation when content is text-heavy with wraps.
