Announcement variant identity — author chip / accent rule / default

Drill resolves the visual identity for the announcement articleType. Path 2 locked 2026-05-18: all announcements (news + columns) render the same; no column-vs-announcement discriminator in the schema, so the lock applies uniformly. The original 4-option set has been narrowed to 3 (Option B from the issue body, "<MonoLabel> COLUMN kicker", required a data signal we don't have — dropped per the data-audit principle).

Same article fixture (3-paragraph piece about training schedule changes) rendered across the 3 treatments. Phase 3-b hero shell + ArticleCredits footer identical across options; only the body identity treatment varies.

Option A · Monogram author chip beside byline
Reuses 5.d2 monogram primitive — zero new data
Small monogram disc (24px) inline with the byline meta row in the hero, derived from article.author[0] — same vocabulary as the locked Q&A row avatars. Personalises every announcement with whoever wrote it. Drops gracefully when author is blank (no chip rendered).
Vocabulary ✓ 5.d2 monogram primitive
✓ article.author (5.d-int addition)
Δ Net-new inline placement (current locked usage: QARow + PullQuote)
