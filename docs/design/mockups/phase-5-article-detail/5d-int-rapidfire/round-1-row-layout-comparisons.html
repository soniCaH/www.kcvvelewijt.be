Rapid-fire QARow — per-row layout (vertical / inline / grid / hanging)

Drill 5.d-int-rapidfire Round 1 of ~4. Locks how a single rapid-fire Q&A pair lays out inside the row. Everything else — speaker scope, row dividers, section opener — is held constant across all four candidates and drilled in later rounds.

All four options share: a single hoisted speaker strip (avatar + tag, top-left) so per-row chrome is minimal; dotted ink-muted dividers between pairs; the prose container (--container-prose: 680px) on the cream surface; and the existing Sneltrein kicker placeholder at the top (kicker treatment is locked separately in Round 4).

Fixture: 4 rapid-fire pairs with a single respondent (Lars Janssens, aanvaller — L). Real prose so the rhythm difference is honest.

Note: per the existing QaGroupRapidFire contract, rapid-fire is intentionally single-respondent. Multi-respondent rapid-fire is explicitly out of scope and will fall back to repeating standard QARows.

Q above A, single column, dotted dividers

Option B · Inline (one line per pair)
Q · A on one row, mono separator dot
Maximum compression. Question and answer share a single line, separated by a mono interpunct. Wraps gracefully when the answer is too long — but only shines when answers stay short. Strongest "rapid-fire" rhythm; weakest when answers run past one sentence.
