## 1. System inheritance (locked in earlier phases — non-negotiable)

These apply identically to the article-detail brief; restated here so each brief is self-contained.

### From Phase 4.5 R9 — Photo treatment system

Applies to **every taped photo on the player profile**: hero portrait, BioBlock pull-quote avatar, QuotesBlock avatars, any "Vier wedstrijden" card imagery if present, all "Verder lezen"-style related-content cards.

- `--filter-photo-newsprint` (sepia 0.06, saturate 0.94, hue-rotate −4deg, contrast 1.02, brightness 1.01) on `<TapedFigure>` images. `data-tint="none"` opt-out for designed graphics.
- Paper grain `::after` overlay: `--pattern-paper-grain` at `opacity: 0.04, mix-blend-mode: multiply`.
- Asymmetric photo shadow: `--shadow-photo-tape` = `2px 4px 0 0 ink`; hover lift = `--shadow-photo-tape-lift`.
- `<TapeStrip edge="torn">` variant + `--color-tape-cream` join `warm` / `jersey` for 3-colour rotation. 1–2 strips per `<TapedFigure>`, independent colour + rotation per strip, slot-deterministic cycle.
- Tape anchors to outer card frame; photo lifts independently on hover.

### From Phase 4.5 R10 — Card structure

Flush-edge cards apply to: "Vier wedstrijden" 4-card grid (if those cards composes `<NewsCard>`-style structure), and any "Verder lezen"-style cards at footer. Outer `<TapedCard>` is the only frame; image (or scoreline) fills the top region edge-to-edge; ink 1px rule divides top region from meta panel.

### From Phase 4.5 R3 — Per-articleType card backgrounds (analogue)

`R3` itself is articleType-driven on `<NewsCard>`. The player profile doesn't render articles, but the same _semantic-tone-by-type_ principle informs:

- **"Vier wedstrijden" card tone:** locked at `tone="ink"` per §5.3 (`<RecentMatchesGrid tone="ink">`). Black bg + cream type + jersey-deep accents.
- **StatsStrip tone:** locked at `tone="ink"` per §5.3.

The cream/jersey-deep/ink palette stays the only tone vocabulary — **no red palette extension without explicit owner sign-off** (relevant to drill Q5 below).

### From Phase 0–4 primitives (master plan §4)

- **`<TapedCard>`** (rotation a/b/c/d, hard offset shadow), **`<TapedFigure>`** (16:9 default; portrait & square variants for portrait crops).
- **`<EditorialHeading>`** with `accent` decorator for jersey-deep emphasis on substring. Italic Freight Display Pro Black at display scales.
- **`<NumberDisplay size="display-2xl" tone="jersey">`** — the canonical large-numeral component. Used here for shirt number AND stats strip numerals.
- **`<MonoLabel>` / `<MonoLabelRow>`** — mono caps labels. `tone="cream"` ships at full opacity per [[feedback_monolabel_cream_full_opacity]] (axe contrast).
- **`<TicketStub>`** — the "season pass" badge primitive. Used here for season label below the player name.
- **`<PlayerFigure imageSrc>`** — photo-first (circular crop via CSS `clip-path`), illustration fallback per [[project_playerfigure_photo_state]] and [[project_playerfigure_illustration_canonical]]. **NOT the hero portrait** — hero is a rectangular `<TapedFigure>`; `<PlayerFigure>` reserved for the smaller avatar / portrait surfaces.
- **`<JerseyShirt>`** — decorative jersey illustration. Used for retro/special-jersey accents; not central to player profile.
- **`<StripedSeam>`** — canonical section divider (legacy `<SectionTransition type="diagonal">` retiring per [[project_diagonal_to_stripedseam_migration]]).
- **`<EndMark flourish="star" | "em-dash">`** — closing flourish.
- **Hover model:** canonical press-down everywhere (`hover:shadow-none hover:translate-x-1 hover:translate-y-1` + `transition-all duration-300`).
- **Founding year:** 1909 per [[reference_club_founding_year]] (relevant if a "since 1909" stamp lands anywhere on this page).

## 6. Schema audit checklist (verified 2026-05-14, before drill rounds open)

Audited `packages/sanity-schemas/src/player.ts` and `<PullQuote>` props during this brief revision. Findings:

- **`player` schema HAS:** `birthDate` (= DoB), `height`, `weight`, `nationality`, `position`, `positionPsd`, `keeper`, `jerseyNumber`, `psdImage`, `transparentImage`, `celebrationImage`, `bio`, `psdImageUrl`.
- **`player` schema LACKS:** `preferredFoot`, `featuredMatches`. Header metadata line drops "preferred foot" by default; featuredMatches defaults to computed.
- **`<PullQuote>` props:** `attribution: { name, role?, source? }` — no separate `attributionDate`. Date folds into `source` as one string.
- **`match` schema:** outside this audit. If `player.featuredMatches` is ever added later, confirm match docs are referenceable from player at that time.

Per [[feedback_design_data_audit]] — never mock fields PSD/Sanity don't provide. The header metadata-row drill mock-up should use **birthDate + height + weight + position + nationality** as the canonical field list, NOT the prompt's "DOB + age + height + weight + preferred foot" (age is computed; preferred foot doesn't exist).
