Icon library comparison.

Phase 2 design checkpoint — visual A/B for the icon strategy. See docs/plans/2026-04-27-redesign-master-design.md §9 open question 8 for context. Lucide is what the site uses today; the redesign direction (retro-terrace fanzine) wants something that sits with cream paper, taped polaroids, and Freight Display italics rather than fighting them.

Open this file in a browser. Icons load from public CDNs (jsdelivr); some may take a moment on first paint. If a square stays blank, the CDN doesn't have that variant — note it and move on; absence in a CDN doesn't necessarily mean the library lacks the icon.

How to read this

  • Lucide column: the current site icons. Reference baseline. Geometric, modern, 2px stroke, perfectly aligned.
  • Phosphor (Fill) column: same shape language as Lucide but solid ink-stamp weight. Cheapest swap from Lucide — the import path is the only change.
  • Game Icons column: hand-drawn engraving feel. Free, CC BY 3.0, ~4000 icons. Skews toward sport / fantasy / medieval but covers most UI primitives too. Best fit for the printed-fanzine vibe. (SVGs ship white-on-transparent so they're CSS-inverted here for visibility on cream — in production they'd be authored or recoloured to ink.)
  • Typographic Glyph column: a Unicode character typed in body or mono font. Zero icon-library payload. Saved owner preference (feedback_visual_preferences.md) already favours this approach where the glyph reads.

Reading the comparison: for the editorial-magazine direction, Phosphor-fill is the safest near-term swap (drop-in compatible API, ink-stamp weight reads as printed). Game Icons wins the brand-character cases (football, trophy, stadium, megaphone, scarf, ticket-stub) where Lucide and Phosphor offer only generic approximations. Typographic glyphs handle the universal quick wins (arrows, stars, checks, asterisks) at zero cost.

Final decision lands at the Phase 2 atom-rework design checkpoint. This comparison is exploratory only.
