# Patterns

Background patterns used by the redesign. The `--pattern-jersey-stripes` and `--pattern-jersey-stripes-tight` tokens are first-class CSS variables — primitives that fill a flat background reference them directly via `background: var(--pattern-...)` rather than redefining gradients inline. The `--pattern-seam` token exists in `globals.css` as a quick-reference for the same pattern, but `<StripedSeam>` itself renders an inline SVG `<pattern>` definition (see `apps/web/src/components/design-system/StripedSeam/StripedSeam.tsx`) rather than consuming the CSS variable, because SVG gives clean start/end caps that a CSS gradient cannot match. New primitives should prefer the CSS-token route when a flat background fill is sufficient and reach for an SVG-backed implementation only when geometry beyond a repeating gradient is needed.

## `--pattern-jersey-stripes`

Wide green/cream horizontal stripes (32 px / 32 px). Used on hero illustrations, dark interlude backgrounds, and the jersey shirt thumbnail in the homepage webshop strip.

## `--pattern-jersey-stripes-tight`

Tight green/cream horizontal stripes (14 px / 14 px). Used on small player figures, retro shirt thumbnails, and background fills where the wide stripes feel coarse.

<div style={{ height: '120px', background: 'var(--pattern-jersey-stripes-tight)', border: '1px solid var(--color-paper-edge)' }} />

## `--pattern-seam`

Diagonal black/cream barber-pole stripe at -45° (8 px / 8 px). Used by `<StripedSeam>` for horizontal section dividers. Never apply with negative margins; `<StripedSeam>` is SVG-backed for clean caps.

<div style={{ height: '24px', background: 'var(--pattern-seam)' }} />

## `--pattern-halftone-dots`

Risograph-style halftone: faint cream dots (~5 % opacity) on an 8 × 8 px grid. Consumers apply it with `background-size: 8px 8px` and `mix-blend-mode: screen`, layered over `--gradient-jersey-deep-overlay` on jersey-deep photo surfaces (e.g. the youth backdrop) to add a printed dot-screen texture.

## `--pattern-paper-grain`

Inline-encoded SVG `<feTurbulence>` noise (Phase 4.5, R9). Applied as a `::after` pseudo overlay on `<TapedFigure>` at 4 % opacity with `mix-blend-mode: multiply` — adds fine paper grain to taped photos so they read as printed rather than digital. The data-URL ships with the stylesheet, no network request. Consumers don't reach for this token directly: it's wired into the `.taped-figure::after` rule in `globals.css`. The preview below shows the noise at a higher 30 % opacity for visibility.

<div style={{ position: 'relative', height: '120px', background: 'var(--color-cream-soft)', border: '1px solid var(--color-paper-edge)', overflow: 'hidden' }}>
  <div style={{ position: 'absolute', inset: 0, backgroundImage: 'var(--pattern-paper-grain)', opacity: 0.3, mixBlendMode: 'multiply' }} />
</div>

## `--filter-photo-newsprint`

CSS filter pipeline applied to `<TapedFigure>` photographs (Phase 4.5, R9). Delivers a "very subtle warm tint" — slight cream shift in highlights, slight green in deepest shadows, like aged newsprint. Composes `sepia(0.06)` `saturate(0.94)` `hue-rotate(-4deg)` `contrast(1.02)` `brightness(1.01)`. Opt out per-instance with `<TapedFigure tint="none">` for designed graphics where the warm cast shifts brand colours.
