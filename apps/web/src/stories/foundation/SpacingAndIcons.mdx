## Icon Catalogue

Icons are [Phosphor](https://phosphoricons.com) glyphs, rendered fill-weight
through the wrapper module `@/lib/icons.redesign`. Each wrapper defaults
`weight="fill"` and accepts `size` (number or string), `className`, `color`,
`aria-label`, and `aria-hidden`. Import the glyph you need by name from that
module — there is no string-name lookup or `<Icon>` wrapper component.

## Redesign / Phase 0 — Layout, Rotation, Shadow, Motion

The redesign introduces four new groups of layout primitives: container widths
(prose / default / index), a four-step rotation pool for taped-card looks,
hard-edged paper shadows for the cream surface, and motion durations for
typical interactions. All values are defined in `src/app/globals.css`.

### Container widths

**These three are the only widths a content container may use.** Page and
section bodies use the shared **`<PageContainer>`** primitive (one centered
container: `mx-auto w-full px-4 md:px-8` + a role-based max-width). Pick by role;
never hand-roll `mx-auto max-w-… px-…`. Heroes and bespoke grid layouts that
can't wrap cleanly in the component may apply the same width **loosely** (the
matching `--container-*` var) — but still only one of these three values:

<table>
  <thead>
    <tr><th>Role</th><th><code>width</code> prop</th><th>Value</th><th>Token / class</th><th>Use for</th></tr>
  </thead>
  <tbody>
    <tr><td>Reading</td><td><code>"prose"</code></td><td>680px</td><td><code>--container-prose</code></td><td>Long-form article / CMS text, forms, legal</td></tr>
    <tr><td>Detail</td><td><em>(default)</em></td><td>1040px</td><td><code>--container-wide</code></td><td>Detail / single-subject pages + hero content</td></tr>
    <tr><td>Index</td><td><code>"index"</code></td><td>1280px</td><td><code>--container-index</code></td><td>Card-grid index / listing / landing pages (incl. homepage)</td></tr>
  </tbody>
</table>

**Not a content container — exempt from the rule above:**

- **Chrome** — `<SiteHeader>` / `<SiteFooter>` use `max-w-[1440px]`. The global nav and footer intentionally span wider than page content; this is the only width above 1280, and it is reserved for chrome.
- **Element-sizing** — a photo, jersey illustration, a paragraph's reading measure / quote / divider / error-message width, or a scaled diagram (e.g. the organigram tree). These size a single visual element, not a page column, and keep their own `max-w-[…]`.
- **Full-bleed** — `<StripedSeam>`, hero band backgrounds, `<CtaBand>`, and coloured section bands span the viewport and are never wrapped.

### Paper shadows

The redesign uses hard-edged offset shadows ("paper shadows") for cards on the
cream surface — they read like risograph print rather than soft web glows. The
soft shadow stays available for chrome that should sit subtly above content
(dropdowns, sticky bars).
