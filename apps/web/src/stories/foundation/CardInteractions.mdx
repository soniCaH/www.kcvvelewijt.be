# Card Interactions

Cards across the site share one tactile idiom from the **retro-terrace fanzine** language: a **square-cornered paper card** (`border-2 border-ink`) sitting on a hard **offset shadow**, that **presses *down* into its shadow on hover**. There is no rounding, no soft blur, and no upward lift. Hover any demo below to feel it.

The primitive is [`<TapedCard>`](?path=/docs/ui-tapedcard--docs); most cards either are a `<TapedCard>` or reuse the same class recipe directly (e.g. `getButtonClasses`).

---

## The press-down model

At rest a card casts a hard, un-blurred offset shadow (`shadow-paper-sm` = `4px 4px 0 0 var(--color-ink)`). On hover the shadow is removed and the card translates `+1`/`+1` (`translate-x-1 translate-y-1` = +4px each) — so it visibly sinks into the space the shadow occupied. This is the single canonical interaction for every paper-stamped surface (cards, buttons, filter tabs, scroll arrows, cookie banner).

## `<TapedCard>` — the card primitive

`<TapedCard>` owns the chrome above and exposes an `interactive` prop:

<table>
  <thead>
    <tr><th><code>interactive</code></th><th>Behaviour</th><th>Used by</th></tr>
  </thead>
  <tbody>
    <tr><td><code>false</code> (default)</td><td>Static paper card, no hover motion</td><td>Info panels, static figures</td></tr>
    <tr><td><code>"press"</code></td><td>The canonical press-down above</td><td>Navigation cards (NewsCard, PlayerCard, …)</td></tr>
    <tr><td><code>"tilt"</code></td><td>Slight photo tilt + scale (hero cover only)</td><td>Homepage `<EditorialHero hoverStyle="tilt-photo">`</td></tr>
  </tbody>
</table>
